# NIST CAVP cross-validation harness

This directory holds a runner for NIST's Cryptographic Algorithm
Validation Program test vectors (the `.rsp` files distributed with
the CAVS suite). The runner parses CAVP-formatted vectors and drives
this SHA3 implementation against each, reporting per-file pass/fail
counts.

The vectors themselves are not bundled — they are NIST-published
artifacts subject to NIST's redistribution terms — and the harness
expects you to fetch them yourself.

## Fetching vectors

NIST's CAVP page for SHA-3:

  https://csrc.nist.gov/projects/cryptographic-algorithm-validation-program/secure-hashing

Download the FIPS 202 vector archive (typically named something like
`sha-3bytetestvectors.zip`) and unpack it. The vectors of interest:

| File                       | Algorithm  | Notes                           |
|----------------------------|------------|---------------------------------|
| `SHA3_256ShortMsg.rsp`     | SHA3-256   | 0-byte through 200-byte msgs    |
| `SHA3_256LongMsg.rsp`      | SHA3-256   | up to several KB                |
| `SHA3_512ShortMsg.rsp`     | SHA3-512   |                                 |
| `SHA3_512LongMsg.rsp`      | SHA3-512   |                                 |
| `SHAKE128ShortMsg.rsp`     | SHAKE128   | fixed 256-bit output            |
| `SHAKE128LongMsg.rsp`      | SHAKE128   |                                 |
| `SHAKE256ShortMsg.rsp`     | SHAKE256   | fixed 256-bit output            |
| `SHAKE256LongMsg.rsp`      | SHAKE256   |                                 |
| `SHAKE128VariableOut.rsp`  | SHAKE128   | each vector has its own Outputlen |
| `SHAKE256VariableOut.rsp`  | SHAKE256   |                                 |

## Running

Build and run from this directory:

```sh
alr build
./bin/cavp_runner sha3_256 /path/to/SHA3_256ShortMsg.rsp
./bin/cavp_runner shake256 /path/to/SHAKE256ShortMsg.rsp
```

A successful run prints a single line per file:

```
SHA3_256ShortMsg.rsp: pass= 137 fail= 0
```

Any failure prints the offending vector's `Len` and expected MD before
the summary line and exits with status 1.

## Limitations

- Monte Carlo (`*Monte.rsp`) test vectors are not yet supported.
- The Variable-Output vectors are honored only insofar as the harness
  reads `[L = N]` headers and per-vector `Outputlen = N` lines and
  sizes the SHAKE output buffer accordingly.

## Why this matters

KAT round-trip tests exercise our own encode/decode against itself,
which confirms internal consistency but cannot detect a misimplemented
permutation that systematically agrees with itself. CAVP vectors are
generated by an independent reference and serve as the contract test
of last resort. A green CAVP run on the published vectors gives us
the same level of cross-validation an FIPS 140-3 lab would require.
